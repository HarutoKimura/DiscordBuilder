# Narration script (English, ElevenLabs-ready)

Paste each block into ElevenLabs separately (one clip per scene) so timing can
be adjusted per cut. Short sentences and commas are deliberate — they read
naturally at TTS pace. Target: calm, confident, ~140 wpm. Total spoken time is
roughly 60 seconds across a 3-minute video; silence over the screen is fine.

| Scene | Time | Line |
|---|---|---|
| 1. Hook | 0:00 | Every community needs small tools. A shift table, an RSVP page, a vote. But nobody builds them. |
| 2. /build | 0:15 | This is DiscordBuilder. You ask for the tool, right where your community already lives. Codex writes the code, in a sandbox, and streams its progress into the thread. |
| 3. Result | 0:45 | A few minutes later: a working app, screenshots, and a link. |
| 4. Multiplayer | 1:15 | The URL is live on the internet. A member signs up for a shift, from their phone. Real backend. Real database. |
| 5. Edit loop | 1:45 | Want a change? Just reply. Codex edits the app — and the data everyone entered survives. |
| 6. Ship gate | 2:20 | And nothing ships until the community agrees. Two thumbs up — and it's released, announced to the whole server. |
| 7. Outro | 2:50 | The conversation is the spec, the history, and the changelog. DiscordBuilder — multiplayer vibe coding. |

## ElevenLabs tips

- Generate scene by scene; regenerate any line whose pacing fights the cut.
- If a line runs long over its scene, cut the middle sentence — each block's
  first and last sentences carry the message.
- Keep one consistent stability/similarity setting across all clips so the
  voice doesn't drift between scenes.
- Export WAV (or highest-quality MP3) and do volume-matching once in the video
  editor, not per clip.
